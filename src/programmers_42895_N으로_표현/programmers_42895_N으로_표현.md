## 💡 프로그래머스 문제
- 문제 링크: [프로그래머스 42895](https://school.programmers.co.kr/learn/courses/30/lessons/42895)
- 문제 요약: 숫자 N을 사칙연산과 이어붙이기(예: 55)만으로 사용해 목표값 number를 만들 때, N 사용 횟수의 최솟값(최대 8회 제한)을 구하는 문제

## 📌 풀이 방법
1. N을 사용한 횟수별로 만들 수 있는 값을 저장하기 위해 Set 배열을 만든다. (dp[1] ~ dp[8])
2. 각 dp[i]에는 N을 i번 이어붙인 값 (예: N, NN, NNN ...)을 먼저 넣는다.
3. i를 1부터 8까지 순회하면서, dp[i] = dp[j]와 dp[i-j]의 값을 사칙연산으로 조합해 채운다.
4. 각 조합에서 number가 나오면 바로 i를 반환한다.
5. 8회까지 시도했는데도 number가 나오지 않으면 -1을 반환한다.

## ✅ 느낀 점
처음에는 모든 경우를 직접 만들어보는 방식이 떠올랐지만, 그렇게 하면 연산 경우가 너무 빠르게 늘어난다는 걸 느꼈다.
N의 사용 횟수별로 만들 수 있는 값을 저장해두고 이를 조합하는 방식으로 접근하니, 불필요한 반복을 크게 줄일 수 있었다.
