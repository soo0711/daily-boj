## 💡 프로그래머스 문제
- 문제 링크: [프로그래머스 42587](https://school.programmers.co.kr/learn/courses/30/lessons/42587)
- 문제 요약: 큐에서 프로세스를 꺼내되, 더 높은 우선순위가 존재하면 다시 뒤로 보내고 그렇지 않으면 실행할 때 특정 위치의 프로세스가 몇 번째로 실행되는지 구하는 문제.

## 📌 풀이 방법
1. 각 프로세스를 (인덱스, 우선순위) 형태의 Node로 만들어 큐에 저장한다. 
2. 큐에서 하나를 꺼낸 뒤, 현재 프로세스보다 더 높은 우선순위가 있는지 확인한다. 
3. 더 높은 우선순위가 존재하면 해당 프로세스를 다시 큐의 뒤로 넣는다. 
4. 없다면 해당 프로세스를 실행하고 실행 순서를 카운트한다. 
5. 실행된 프로세스는 priorities 배열에서 제거 표시(-1)로 갱신한다. 
6. 실행된 프로세스의 인덱스가 location과 같다면, 현재까지의 실행 순서를 반환한다.

## ✅ 느낀 점
단순 큐 문제라고 생각했는데, 우선순위를 계속 비교해야 해서 매번 전체를 순회하는 방식이 비효율적일 수 있다는 걸 느꼈다. 
정렬이나 우선순위 큐 활용해서 다시 풀어봐야겠다.
