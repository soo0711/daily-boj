## 💡 프로그래머스 문제
- 문제 링크: [프로그래머스 12911](https://school.programmers.co.kr/learn/courses/30/lessons/12911)
- 문제 요약: 자연수 n보다 크면서 2진수에서 1의 개수가 동일한 수 중 가장 작은 값을 찾는 문제.

## 📌 풀이 방법
1. Integer.bitCount()를 활용해 n의 1의 개수를 먼저 구한다. 
2. n+1부터 시작해서 값을 1씩 증가시키며 반복한다. 
3. 각 숫자의 1의 개수를 구해 n과 같은지 비교한다. 
4. 조건을 만족하는 순간의 값을 바로 반환한다.

## ✅ 느낀 점
처음에는 비트 연산으로 직접 1의 개수를 세는 방법을 떠올렸지만 Integer.bitCount() 같은 내장 함수를 활용하면 훨씬 간결하고 실수를 줄 일 수 있었다.
단순 반복문이라도 문제의 입력 범위를 기준으로 시간복잡도를 판단하는 게 중요하다는 걸 다시 한번 확인할 수 있었고 최악의 경우에도 실제 탐색 범위가 제한적이라면 충분히 효율적인 풀이가 될 수 있다는 점을 배웠다.
