## 💡 프로그래머스 문제
- 문제 링크: [프로그래머스 17680](https://school.programmers.co.kr/learn/courses/30/lessons/17680)
- 문제 요약: 도시 이름 요청을 LRU 캐시로 처리할 때, 캐시 크기에 따른 cache hit/miss 비용을 계산해 총 실행시간을 구하는 문제.

## 📌 풀이 방법
1. HashMap<String, Integer>를 사용해 도시 이름과 최근 사용 시점을 저장한다. 
2. 도시 이름은 대소문자를 구분하지 않으므로 toLowerCase()로 통일한다. 
3. 캐시 크기가 0이면 모든 요청이 miss이므로 cities.length * 5를 바로 반환한다. 
4. 반복문을 돌면서 현재 도시가 map에 존재하면 cache hit으로 간주하고, 해당 도시의 사용 시점을 현재 시간으로 갱신하며 실행시간을 1 증가시킨다. 
5. 존재하지 않으면 cache miss로 처리하고 실행시간을 5 증가시킨다. 
6. 캐시가 아직 꽉 차지 않았다면 그대로 map에 추가한다. 
7. 캐시가 가득 찬 상태에서 miss가 발생하면, map에 저장된 값(사용 시점)이 가장 작은 데이터를 찾아 제거(LRU)하고 새로운 도시를 추가한다. 
8. 최종적으로 누적된 실행시간을 반환한다.

## ✅ 느낀 점
LRU 캐시를 직접 구현해보면서 단순 개념으로만 알던 “가장 오래 사용되지 않은 데이터 제거” 방식을 실제 코드로 어떻게 표현하는지 이해할 수 있었고, 캐시 동작 원리에 대해 더 확실히 익힐 수 있었다.
