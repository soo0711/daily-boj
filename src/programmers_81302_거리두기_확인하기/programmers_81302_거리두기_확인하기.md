## 💡 백준 문제
- 문제 링크: [프로그래머스 81302](https://school.programmers.co.kr/learn/courses/30/lessons/81302)
- 문제 요약: 5×5 대기실 5곳에서 응시자(P)들이 파티션(X)을 고려한 맨해튼 거리 2 이하 거리두기 규칙을 모두 지키는지 검사해, 각 대기실마다 준수 여부(1 또는 0)를 판단하는 문제

## 📌 풀이 방법
1. 대기실은 총 5개이므로 각 대기실을 하나씩 순회한다.
2. 각 대기실은 문자열 배열로 주어지기 때문에, 좌표 접근을 쉽게 하기 위해 `char[5][5]` 형태의 2차원 배열로 변환한다.
3. 대기실 내 모든 좌표를 탐색하며 응시자(`P`)가 있는 위치를 찾는다.
4. `P`를 발견하면 해당 좌표를 기준으로 **맨해튼 거리 1**인 위치를 먼저 검사한다.
    - `(x + 1, y)`, `(x, y + 1)`
    - 인접한 위치에 `P`가 존재하면 즉시 거리두기 위반으로 판단한다.

5. 다음으로 **맨해튼 거리 2**인 위치를 검사한다.
    - **직선 거리**
        - `(x + 2, y)` → 중간 위치 `(x + 1, y)`가 `X`이면 허용
        - `(x, y + 2)` → 중간 위치 `(x, y + 1)`가 `X`이면 허용
    - **대각선 거리**
        - `(x + 1, y + 1)`, `(x + 1, y - 1)`
        - 두 방향 중 하나라도 파티션(`X`)이 없으면 거리두기 위반으로 판단한다.
6. 거리두기 위반이 발생하면 해당 대기실 탐색을 즉시 종료하고 결과를 `0`으로 저장한다.
7. 대기실 내 모든 `P`가 조건을 만족하면 해당 대기실은 거리두기를 지킨 것으로 판단하여 `1`을 저장한다.
8. 모든 대기실에 대한 결과를 배열에 담아 반환한다.


## ✅ 느낀 점
- 처음에는 모든 P 좌표를 저장해서 서로 비교하려 했지만, 경우의 수가 많아지고 로직이 복잡해졌다.
- 맨해튼 거리 2 이하의 경우가 정해진 패턴으로 한정된다는 점을 이용하니 훨씬 단순하게 풀 수 있었다.
