## 💡 프로그래머스 문제
- 문제 링크: [프로그래머스 42628](https://school.programmers.co.kr/learn/courses/30/lessons/42628)
- 문제 요약: 삽입과 최댓값/최솟값 삭제 연산이 반복되는 이중 우선순위 큐를 구현한 뒤, 마지막에 남은 최댓값과 최솟값을 반환하는 문제.

## 📌 풀이 방법
1. 최솟값을 꺼내기 위한 최소 힙과 최댓값을 꺼내기 위한 최대 힙을 각각 생성했다.
2. 삽입 연산이 들어오면 두 우선순위 큐에 모두 값을 넣고, HashMap에 해당 숫자의 실제 개수를 저장했다.
3. 최솟값 삭제 연산일 경우 최소 힙의 top이 이미 삭제된 유령 값이면 제거를 반복한 뒤, 유효한 값을 하나 꺼내 개수를 감소시켰다.
4. 최댓값 삭제 연산도 최대 힙에서 동일하게 유령 값을 정리한 후, 유효한 값을 삭제 처리했다.
5. 두 우선순위 큐는 서로 직접 동기화되지 않기 때문에 삭제된 값이 반대쪽 큐에 남아 있을 수 있어, Map을 이용한 lazy deletion 방식으로 실제 유효 여부를 판단했다.
6. 모든 연산이 끝난 뒤에도 각 큐의 top에 유령 값이 남아 있을 수 있으므로 마지막으로 한 번 더 정리한 후 최댓값과 최솟값을 반환했다.

## ✅ 느낀 점
우선순위 큐 여러 개를 사용할 때는 단순히 값을 넣고 빼는 것보다 각 자료구조의 상태를 어떻게 동기화할지 고민하는 것이 중요하다는 점을 배웠다.
