## 💡 프로그래머스 문제
- 문제 링크: [프로그래머스 12941](https://school.programmers.co.kr/learn/courses/30/lessons/12941)
- 문제 요약: 자연수 n을 연속된 자연수들의 합으로 표현할 수 있는 경우의 수를 구하는 문제

## 📌 풀이 방법
- 투포인터 방법
1. 연속된 자연수의 합을 구해야 하므로 투 포인터를 활용해 구간을 이동하며 합을 계산했다.
2. start, end 포인터를 사용해 연속된 구간을 만든다. 
3. 현재 합이 n보다 작으면 end를 증가시켜 구간을 확장한다. 
4. 현재 합이 n보다 크면 start를 증가시켜 구간을 축소한다. 
5. 합이 n과 같다면 경우의 수를 증가시키고 start를 이동시킨다.
6. 시간복잡도 O(n)

- 최적화 방법
1. 연속된 자연수의 합은 n = a + (a+1) + ... + (a+k-1) 이다.
2. 이를 정리하면 2n = k(2a + k - 1)
3. 이 식을 통해 연속된 자연수의 개수 k는 n의 홀수 약수와 대응된다는 것을 알 수 있다.
4. 따라서 n의 홀수 약수의 개수를 구하면 정답을 구할 수 있다.
5. 시간복잡도는 O(√n)

## ✅ 느낀 점
연속된 구간 문제라 자연스럽게 투 포인터로 접근했지만
수열의 규칙성을 이용하면 수학적으로 더 빠르게 해결할 수 있다는 걸 알게되었다.
문제를 여러방면에서 고민해보고 풀어나가야겠다.
