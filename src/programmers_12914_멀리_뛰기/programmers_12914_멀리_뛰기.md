## 💡 프로그래머스 문제
- 문제 링크: [프로그래머스 12914](https://school.programmers.co.kr/learn/courses/30/lessons/12914)
- 문제 요약: 1칸 또는 2칸씩 이동해 총 n칸에 도달하는 모든 경우의 수를 구하는 문제.

## 📌 풀이 방법
1. n칸에 도달하는 경우를 생각할 때, 마지막에 1칸을 뛰었는지 2칸을 뛰었는지로 나눌 수 있다.
2. 예를 들어 3칸은 (1칸에서 +2) 또는 (2칸에서 +1)로 만들 수 있으므로 dp[3] = dp[1] + dp[2]가 된다.
3. 마찬가지로 4칸은 (3칸에서 +1)과 (2칸에서 +2) 경우를 더해 dp[4] = dp[3] + dp[2]로 표현된다.
4. 현재 칸의 경우의 수는 이전 두 칸의 경우의 수를 더한 값이므로 점화식 dp[n] = dp[n-1] + dp[n-2]를 사용한다.
5. 초기값 dp[1] = 1, dp[2] = 2를 설정하고 반복문으로 n까지 계산하며, 매번 1234567로 나눈 나머지를 저장한다.

## ✅ 느낀 점
처음에는 DFS로 모든 경우의 수를 구하려 했지만 n이 최대 2000이라 시간초과가 발생했다. 
작은 값부터 직접 경우의 수를 구해보니 이전 결과를 활용하는 점화식을 도출할 수 있었고, 이를 통해 효율적으로 해결할 수 있었다. 
앞으로는 무작정 완전탐색을 시도하기보다, 입력 크기를 보고 시간복잡도를 먼저 고려하는 습관을 가져야겠다고 느꼈다.
